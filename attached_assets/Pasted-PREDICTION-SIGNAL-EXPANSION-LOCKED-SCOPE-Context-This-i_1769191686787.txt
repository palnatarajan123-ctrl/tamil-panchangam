PREDICTION SIGNAL EXPANSION (LOCKED SCOPE)
Context:
This is a FastAPI-based South Indian astrology engine already deployed on Replit.
Core components already exist:
•	Birth chart (Rasi)
•	Navamsa (D9)
•	Mahadasha + Antar Dasha
•	Monthly prediction pipeline (envelope → synthesis → interpretation)
The current issue is that predictions are static and repetitive because only slow-moving signals (Dashas) are used.
Your task is to add missing, canonical South Indian astrology prediction signals while keeping architecture clean, deterministic, and auditable.
 
🎯 OBJECTIVE
Implement motion-based and quality-based prediction signals so that:
•	Monthly / weekly predictions actually change
•	Life-area text reflects real astrological movement
•	AI (later) will only explain already-derived signals
⚠️ Do NOT introduce generative AI yet. Deterministic only.
 
🔒 HARD CONSTRAINTS (DO NOT BREAK)
1.	Do not change existing database schemas unless explicitly required.
2.	Do not remove or refactor existing Dashas, LifeAreaScorer, or synthesis logic.
3.	New logic must live in new engine modules, not inside existing ones.
4.	Every new signal must be:
o	Deterministic
o	Explainable
o	Logged (debug-friendly)
